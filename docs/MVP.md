# Cofree 当前交付范围（v0.0.8）

本文件不再描述“计划中的 MVP”，而是直接说明 **当前版本已经交付并可验证的使用范围**。

## 1. 一句话定义

给定一个本地工作区，Cofree 目前可以完成：

**需求输入 -> 读取上下文 -> 生成 patch 或 shell 动作 -> 人工审批 -> 执行 -> 展示结果与诊断 -> 记录审计与恢复信息**

## 2. 当前黄金路径

### 2.1 从需求到代码修改
前置条件：
- 已选择本地工作区。
- 已配置可用模型连接与 API Key，或指向本地模型网关。

流程：
1. 用户输入需求。
2. 系统读取工作区上下文并搜索相关文件。
3. 系统生成待审批 patch。
4. 用户在 diff 视图中批准、拒绝或补充意见。
5. 批准后系统应用 patch，并自动执行工作区诊断。
6. 结果写入聊天记录、工具追踪和审计日志。

### 2.2 从需求到命令执行
1. 用户输入需要构建、测试或 Git 操作的需求。
2. 系统生成待审批 shell 动作。
3. 用户审批完整命令。
4. 系统在工作区目录执行命令并返回 stdout / stderr / exit code。

### 2.3 拒绝与恢复
1. 用户拒绝某个 patch 或 shell 动作。
2. 系统不执行对应副作用。
3. 用户可继续追问或补充评论，进入下一轮方案生成。
4. 已通过审批但执行失败的 patch 可从快照回滚；工作流状态可从 checkpoint 恢复。

## 3. 当前范围内的能力

### 3.1 必备能力
- 代码搜索、文件读取、Git 状态/差异读取。
- patch 审批、批量审批、shell 审批。
- patch 预检、失败回滚、自动诊断。
- LLM 请求和敏感动作审计。
- 工作流 checkpoint 恢复。

### 3.2 辅助能力
- 多会话管理，且按工作区隔离存储。
- 顶层 ChatAgent 与内部 Sub-Agent。
- 工具权限的 `ask` / `auto` 模式。
- `.cofreerc` 项目配置加载。
- 控制台页查看统计、时间线、审计导出。

## 4. 明确不纳入当前范围的内容

- 强隔离系统沙箱。
- hunk 级或行级独立审批。
- 专用 Git commit 向导。
- 自动 push 到远端。
- 复杂 Git 场景的自动修复。
- 并行 Sub-Agent 或多工作区并行执行。

## 5. 当前验收基线

以下行为应被视为“当前产品应稳定提供”的基线：

- 未审批前，默认不会执行写文件或 shell 命令。
- 用户能看到完整 diff 或完整命令后再做决定。
- patch 应用后可拿到自动诊断结果。
- shell 执行后可看到 stdout / stderr / exit code。
- 用户能在控制台页看到工具追踪和审计记录。
- 应用重启后可以恢复最近的工作流 checkpoint。
