# Cofree Security & Privacy（v0.1）

本文件用于把“100% 本地 + 自带 API Key”落实为可验证的安全与隐私规范。

## 1. 术语
- **本地执行（Local Execution）**：文件读取、patch 生成、diff 渲染、git 操作、日志存储都在用户机器本地完成。
- **模型调用（LLM Request）**：向模型提供商（或本地模型）发送请求并接收响应。
- **数据外泄（Data Egress）**：任何离开本机的内容（包括代码片段、diff、文件路径、命令输出、错误栈等）。

## 2. v0.1 的隐私承诺（必须满足）
1) Cofree 不会在未经用户同意的情况下上传代码仓库。
2) 所有“敏感动作”（写盘/命令/git 写操作）必须经过审批门。
3) 所有模型调用必须被记录到本地审计日志（不记录 API Key；对内容做截断/摘要）。

## 3. Data Egress Policy（必须写清楚、可实现）

### 3.1 允许发送的数据类型（默认）
- 用户输入的自然语言需求
- 为完成任务所需的最小代码上下文（片段级），以及必要的 diff（建议摘要/截断）
- 结构化计划与工具调用意图（不包含秘密）

### 3.2 默认禁止发送的数据类型
- `.env`、密钥文件、token、证书、SSH key、Keychain 内容
- 整个仓库打包上传
- 大体量文件内容（除非用户显式批准）

### 3.3 用户控制（v0.1 最低要求）
- 用户可在设置页选择：
  - 仅使用本地模型 / 允许云模型
  - 发送代码片段的最大长度（例如 200/500/2000 行）
  - 是否允许发送文件路径（默认：只发送相对路径或脱敏路径）

## 4. API Key 存储（v0.1）
- 禁止将 API Key 写入 git 可跟踪文件。
- 建议：macOS Keychain（若实现成本高，至少保证写入本地配置文件时加密/权限受控）。

## 5. 审计日志（Audit Log）
### 5.1 必须记录的事件
- LLM 请求：provider/model、时间、输入输出长度、是否含代码片段（布尔）、请求 id
- 任何写盘：目标路径（相对）、字节数、结果
- 任何命令执行：命令字符串、工作目录、结果（exit code）
- 任何 git 写操作：branch、commit id、结果

### 5.2 日志内容约束
- 不记录 API Key
- 内容字段应截断或存摘要（hash），避免把敏感数据写入日志

## 6. 安全边界声明（v0.1）
- v0.1 的“安全”主要来自 guardrails 与审批门，不等同于强隔离 sandbox。
- 若用户允许执行命令或写盘，风险由用户审批承担；系统必须提供清晰的预览与提示。
