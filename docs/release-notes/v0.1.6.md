## Cofree v0.1.6

> **本地优先的 AI 编程助手**——你提需求、看审批、做验收，AI 负责读代码、生成 patch、执行命令。

本次发布带来一个轻量但实用的能力：**知识 / Snippets**。它和 Skill 长得很像，但只在你 `@` 显式选中时才把内容注入到当轮系统提示中——绝不会基于消息文本自动激活。适合放进 HDY 服务器、内部 API、约定俗成的运维口径之类「需要时才告诉模型」的背景知识。

---

## ✨ 亮点

### 📑 知识 / Snippets（新功能）
- **绝不自动激活**：不像 Skill 会基于关键词 / 文件模式 / 通用意图回退自动匹配，Snippet 仅在你于聊天框输入 `@` 选中后注入。未选中时它对模型完全不可见。
- **`@` 选择器同列表分组**：picker 内 Skill / 知识 / 文件混排，知识用 `📑` 图标和 emerald 色 chip 区分，便于一眼挑选。
- **每条独立的 .md 文件**：磁盘存储路径为 `~/.cofree/snippets/<name>.md`，YAML frontmatter 保存 `name` / `description`，正文为注入的字面文本。也支持「仅本应用」的纯自定义条目（不写盘）。
- **跨轮一致性**：消息记录里持久化 `explicitSnippetIds[]`，HITL 续跑、消息编辑重放都会自动恢复同一组 snippet 上下文，模型不会在多轮中途突然失去你刚附加的背景。
- **设置页全 CRUD**：「知识 / Snippets」tab 支持新增 / 编辑 / 启用禁用 / 删除，描述字段为必填（用作 picker 列表的副标题）。
- **注入格式**：每个激活的 snippet 以 `<snippet name="...">...</snippet>` 块形式追加在 system prompt 末尾，标题层级零污染，便于模型识别边界。

### 🔧 后端
- 新增 Tauri 命令 `write_snippet_file` / `delete_snippet_file`，严格限制在 `~/.cofree/snippets/` 目录下；文件名仅允许 ASCII 字母数字、下划线、短横线。
- `assembleSystemPrompt` 接受可选的 `snippetResolution`，与 skill fragment 解耦串联。

---

## 🔍 升级须知
- 本次新增 `AppSettings.snippets[]` 字段；旧版本设置自动在加载时归一化为 `[]`，无需手动迁移。
- 没有 snippet 时行为完全不变——这是新功能，不影响既有 skill / 文件附加流程。
