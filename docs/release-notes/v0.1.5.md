## Cofree v0.1.5

> **本地优先的 AI 编程助手**——你提需求、看审批、做验收，AI 负责读代码、生成 patch、执行命令。

本次发布以 **上下文工程（Context Engineering）** 与 **对话可控性** 为主线：上线 Anthropic 提示缓存、跨轮文件内容缓存与陈旧 read_file 自动去重，长会话的 token 成本显著下降；同时新增「编辑历史用户消息并重放」的对话编辑能力，重写代码仓符号提取以更准确地为 LLM 描述工作区。

---

## ✨ 亮点

### 🧠 上下文引擎重构（M1 / M3 / X 系列）
- **Anthropic 提示缓存**：在 system prompt 设置 `cache_control` 锚点，并通过审计日志/控制台/CSV 导出暴露缓存命中的 token 计数（OpenAI 自动缓存同样可见）。
- **稳定的固定槽顺序（pinned slots）**：固定头部槽位按确定顺序拼接，跨轮保证字节级一致，是缓存命中的前提。
- **文件内容跨轮缓存**：`read_file` 的内容写入 `WorkingMemory.fileKnowledge[path].content`，仅在内容真正变更时升版本号；`serializeWorkingMemory` 会在工作记忆里标注 `✓ 内容已缓存 v{N}`，提示模型无需重读。
- **陈旧 read_file 自动去重**：每轮对话结束后，过往同一路径的 `read_file` 结果会被替换为 JSON 桩；若槽位被失效（例如 `propose_file_edit` 即将覆盖该文件），所有历史副本（含最新一份）都会被替换为「内容已过期」提示。
- **压缩决策大幅简化**：删除 `compressionScheduler`、`MessageTokenTracker`、模型级 EMA 校准等冗余层；`smartTruncate` 改为「保留尾部」策略并在标记里报告丢弃的字符/行数；`ensureUserPresence` 转为硬断言。整体净减约 **461 行代码**，行为等价或更优。

### 🗺️ 代码符号提取（M4）
- **TS / JS / Vue / Svelte 模式重写**：将原本只识别 `export` 关键字的正则集，重写为 6 类分别匹配 class / interface / enum / type / function / constant 的 kind-aware 正则。新增对 `export const Foo = () => ...` 箭头导出、`export default class`、`export async function`、生成器、抽象类、顶层 `interface` / `enum` 的覆盖。
- **类成员提取**：通过括号深度状态机识别类体内方法签名，覆盖 `public/private/static/async/get/set` 与泛型 `name<T>(...)`，每个类最多 8 个方法，关键字黑名单（`if/for/while/return` 等）过滤误报。Java / Kotlin 改走同一路径，方法只在类作用域内匹配。
- **签名头剪裁**：`signature_head` 在第一个 `\u{20}{` 或 `\t{` 处截断函数体，但能区分模板字符串 `${...}` 中的花括号；`export function gatewayComplete(opts): Promise<R> { ... }` 现在以 `export function gatewayComplete(opts): Promise<R>` 形式存入仓库地图。
- **前端格式修正**：`formatSymbolCompact` 现在按 kind 输出正确的单字符前缀（`c/i/t/v/m/f`），不再把所有匹配都归为 `e:`。
- TS 主项目下符号覆盖率预期提升 **3–5 倍**，为后续的「按需注入文件骨架」打下基础。

### ✏️ 编辑历史用户消息并重放（M5）
- **可编辑的用户气泡**：用户消息悬停展示 ✎ 按钮（流式回复中或当前正在编辑时不显示），点击后将该消息内容载入到输入框。
- **截断 + 重放**：提交编辑后，会话从该消息处截断、刷盘、清除该会话的所有 checkpoint 与缓存的工作记忆快照，再以新内容启动一次完整的 planning 周期。
- **错误回滚**：在新一轮 planning 真正开始前出错，会回滚消息列表并保留编辑态，便于直接重试。
- 新增 9 个测试覆盖 `truncateMessagesFrom`、ChatComposer 编辑态与气泡编辑按钮。
- 注：本次未保留分支历史（编辑后旧子树会被丢弃），保留分支需要 v2 数据模型，留作后续。

### 🐛 修复
- **聊天气泡溢出滚动条**：用户消息行的右侧编辑按钮（`right: -28px`）此前会让 `.chat-thread` 出现横向滚动条；现在为该行预留 `padding-right: 32px`。

## 🔍 升级须知
- 本次 LLM 缓存改造同时影响 Anthropic 与 OpenAI 路径；如果在私有网关上自行做了缓存策略，请在升级后核对 audit log 中的 `cache_creation_input_tokens` / `cache_read_input_tokens` 字段是否符合预期。
- HITL 审批通过的 patch（区别于工具自动执行）仍在 planning loop 之外应用，因此跨会话恢复时仍可能携带过期文件内容；运行期会通过 `propose_file_edit` 提交时的预投机失效（speculative invalidation）覆盖大部分场景，残余情况留待后续会话级 mtime 校验。
